# CLOUD COMPUTING - PROJECT-BASED ASSESSMENT

## Assignment Title

Design, Implementation, and Deployment of a Cloud-Based Customer Contact Management SaaS on AWS

## Unit Description

This project-based assessment evaluates students' understanding and practical skills in cloud computing, focusing on cloud architecture design, service configuration, external cloud integration, and Software-as-a-Service (SaaS) deployment using Amazon Web Services (AWS).

Students are required to design, implement, and deploy a cloud-based web application for managing customer contacts and communicating with customers via SMS and email.

## Assessment Type

- Project-Based Assessment
- Group

## Learning Outcomes Assessed

Upon successful completion of this assignment, students will be able to:

- Design a cloud architecture using AWS services.
- Develop and deploy a web application on the cloud.
- Configure and integrate external cloud services.
- Implement a scalable and secure SaaS solution.
- Demonstrate understanding of cloud deployment models and service models.

## Project Overview

You are required to design, implement, configure, and deploy a cloud-based web application that manages customer contact information and supports customer communication through SMS and email.

The system must be deployed as a Software-as-a-Service (SaaS) solution on AWS Cloud.

## Project Requirements

### 1. Web Application Development

Implement a web application with the following features.

#### Core Functionalities

- Manage customer contact information:
  - Full Name
  - Address
  - Phone Number
  - Email Address
- Create, Read, Update, Delete (CRUD) customer records.
- User-friendly web-based interface (UI).

#### Communication Features

- Send SMS to customers.
- Send Email to customers.
- Allow selection of individual or multiple customers for communication.

### 2. AWS Cloud Architecture Design

Design and configure an AWS-based architecture for the application.

Your architecture should include (but is not limited to):

- Compute service (for example: EC2, Elastic Beanstalk, or ECS).
- Storage service (for example: RDS, DynamoDB, or S3).
- Networking (VPC, Subnets, Security Groups).
- IAM roles and permissions.
- Monitoring and logging (for example: CloudWatch).

Deliverable:

- Cloud architecture diagram.
- Explanation of selected AWS services and their roles.

### 3. External Cloud Service Integration

Register and configure at least one external cloud service, such as:

- SMS service (for example: Twilio, AWS SNS, or similar).
- Email service (for example: AWS SES, SendGrid, or similar).

Students must:

- Configure API credentials securely.
- Integrate the service into the web application.
- Demonstrate successful SMS and/or email delivery.

### 4. SaaS Implementation and Deployment

Deploy the application as a SaaS solution on AWS.

Requirements:

- Application must be accessible via a public URL.
- Cloud deployment must support scalability.
- Basic security practices must be applied.
- Environment configuration must be cloud-based (not local).

## Technical Requirements

- Cloud Platform: Amazon Web Services (AWS)
- Application Type: Web Application
- Programming Language / Framework: Open choice (subject to lecturer approval)
- Database: Cloud-based
- External Services: SMS and/or Email provider

## Deliverables

Students must submit the following:

1. Project Report (PDF)
   - System overview
   - AWS architecture design and explanation
   - External cloud service configuration
   - SaaS deployment process
   - Security considerations
   - Screenshots of application and deployment
2. Source Code
   - Clean and well-documented
   - Repository link (for example: GitHub)
3. Live Demonstration

   - Demonstration of:
     - Customer management
     - SMS/email sending
     - Cloud deployment
